指标体系

North Star

Healthy D30 Retention — 自发回访，非推送驱动，按年龄段分层

安全指标（4层）

曝光层 有害内容曝光率、兔子洞深度、首次有害曝光时间
响应层 举报率、会话放弃率、安全功能激活率
结果层 骚扰举报率变化、账号删除率、监管投诉率
系统层 AUC（红队）、申诉推翻率、漏审率
陷阱：举报率↓ ≠ 骚扰减少，需同时看"举报后行动率"
合规产品指标（P0-P4）
P0CSAM上报合规率 = 100%，法律红线，24h内上报NCMEC
P0COPPA数据合规率 = 100%，U13 数据不用于广告定向
P1有害内容曝光率 每季度下降，按U13/13-15/16-17分层
P1陌生人接触率 仅好友政策上线后下降≥50%
P2Opt-out率 <20%，>60%说明产品设计有问题
P2家长满意度NPS >30，监管信任的领先指标
P3人审产能利用率 70-90%，>95%审核员疲劳
P4合规成本效率 每季度下降，DSA罚款最高全球营收6%
Score 阈值计算（10人团队）

三个约束找交集

产能约束 每日送审量 ≤ 10,000（10人×1000条）
Recall约束 ≥ 90% → 阈值上限，不能太高
Precision约束 ≥ 15% → 阈值下限，不能太低
选法 交集里选 Precision 最高的点
校准 每周千人标样，漏审率>5%下调阈值

高分段样本不足时

合并相邻 bin（70-80 + 80-90 → 70-90）
Bootstrap 置信区间估计
等待积累（临时保守阈值保底）
高分段即使CI宽，有害率下限仍高，可以直接拍板

人力成本公式

额外人数 = (目标送审量 - 当前产能) / 1000
月成本 = 额外人数 × $200 × 30
AUC / F1 / P / R 用途
AUC 比较模型版本，不依赖阈值。随机取有害vs无害，排对的概率
Recall 定阈值上限，安全红线 ≥ 90%。漏掉有害=最大风险
Precision 定阈值下限，人审效率 ≥ 15%。太低→疲劳→漂移
F1 模型迭代参考基准，不是决策指标
F1最优点 ≠ 业务最优点，因为Recall有强约束
Ground Truth 问题
覆盖偏差 只能评估模型发现的，漏掉的永远看不到
红队种子集 已知有害内容测召回率，每月更新
随机抽低分段 每周1000条人审，估算漏审率
捕获-再捕获 N=(n_A × n_B)/n_AB，两源独立时有效
PU Learning 已举报=正，未举报=未知（不是负）
灵敏度调整 真实率=观测率/召回率，消除A/B两组检测差异
ML 内容过滤流水线

层1：多模态分类

视频ViT + Wav2Vec2（音频）
Whisper→BERT（语音转文字）
TrOCR（画面文字）
融合：late / early / cross-attention
输出：风险分数0-100（非二元）

层2：用户上下文

年龄推断：XGBoost（行为+设备+社交图谱）
高精确率保守阈值，人工复核边界case
年龄≠报告年龄，模型推断更可靠
不同年龄段应用不同风险阈值

层3：推荐重排序

硬过滤：高置信有害内容直接移除
惩罚重排：score - λ×risk_score
λ 按年龄段递增（U13最大）
长期目标：独立青少年排序模型
人审工作流（Feed Pool / CSAT / CSAM）

Feed Pool 1000条设计

每个国家单独抽样（文化/语言/监管不同）
分层：分数区间 + 内容类型 + 高风险类别超采样
低分段多抽（捕捉漏审）、中分段重点审（边界）
CSAT审核员看：内容+账号信息+模型分数+评论区

审核质量监控

IAA（Cohen's Kappa）> 0.8 为强一致
审核员漂移检测：每周线性趋势分析
新审核员校准期：2周复核 + Kappa>0.75才独立上岗

CSAM 强制上报

法律要求 美国24h内上报NCMEC，违规最高$150,000/次
技术 PhotoDNA hash 匹配（不是ML分类）
流程 检测→立即删除→封号→上报NCMEC
监控 上报延迟P99、hash覆盖率、误报率
唯一没有recall/precision权衡的场景，目标零漏报

CGVR（诱骗风险）

单条内容可能无害，需看账号行为序列
用 LSTM/Transformer 检测成年账号异常互动模式
特征：与未成年人互动占比、个人信息索取频率、平台外接触尝试
A/B 测试设计要点

Pre-feasibility 4维度

技术 指标如何定义？工程实现无障碍？
样本量 功效分析→实验周期可行？
法律 控制组合规？需Legal预审
伦理 两组都有基础保护，对比程度不对比有无

随机化设计

单位 family_id（不是user_id）
原因 家长孩子必须同组，否则投诉数据被污染
分层 年龄段 + 使用强度 + 地区
时长 ≥4周（避免novelty effect）
停止条件 DAU下降>5%立即停止

指标陷阱

accept_rate 只存在实验组，不能跨组比较
→ 只用于实验组内优化弹窗设计
正确主指标 两组都能测的行为指标
如：自愿会话结束率、60分钟后继续使用率
D30指标 入组期结束后再等30天观测，常被忽略
功效分析 + CUPED

四参数口诀

α=0.05 假阳性率，安全场景可放宽到0.10
功效=0.80 真实效果被检测到的概率
MDE 先和PM对齐，再算样本量（最重要）
N公式 (Z_α/2+Z_β)² × [p1(1-p1)+p2(1-p2)] / MDE²
总周期 入组期 + 观测期 + 14天novelty缓冲

样本不足时

放宽MDE / 降低触发门槛 / 换代理指标
分阶段实验 / 历史数据回溯

CUPED 方差缩减

公式 Y_cuped = Y - θ(Y_pre - E[Y_pre])
θ 必须用实验前数据估计，不能用实验期
效果 方差缩减 = ρ²（相关系数的平方）
ρ=0.7 → 缩减49%，实验周期减半
TikTok典型值 D30 retention ↔ 实验前D30: ρ≈0.7-0.8
θ用实验期数据 → 被处理效应污染 → 错误结论
合规因果推断方法

DiD

有控制组 + 平行趋势成立
验证：画8-12周预处理趋势
分阶段rollout → Callaway-Sant'Anna
安慰剂测试：假处理日期系数≈0

回归断点（RD）

年龄断点：13岁/18岁（天然实验）
用户无法操控生日 → 内生性很小
CCT最优带宽选择
McCrary检验：密度在断点连续

面试金句速查

阈值 "三个约束的动态平衡：产能给下限，Recall给上限，Precision保效率"
AUC "随机取一个有害和一个无害，模型排对的概率"
Ground Truth "覆盖偏差问题——模型漏掉的永远看不到"
CUPED "θ必须在实验前固定，用实验期数据会污染处理效应"
Family Pairing "以family_id为随机化单元，同一家庭所有账号分到同一组"
RD设计 "用户无法操控生日，年龄断点是最干净的自然实验"
Accept rate "只存在实验组，不能跨组比较，只用于弹窗设计优化"
CSAM "唯一没有recall/precision权衡的场景，目标零漏报"
CGVR "单条内容无害，行为序列有害——用账号级序列模型"
合规指标 "约束优化问题，不是单一最大化——P0红线不可妥协"
