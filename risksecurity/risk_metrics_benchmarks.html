System Health Metrics

指标健康范围警戒线紧急阈值异常时诊断方向数据来源
Model AUC
P3
（红队种子集，每周）		 整体: >0.92
CSAM类: >0.99
边界内容: >0.85
周周稳定		 周环比下降>0.02 整体下降>0.05
CSAM AUC <0.98		 按内容类别拆AUC → 哪类下降最多？
检查红队种子集是否过期（需每月更新）
数据分布漂移检测 → 新型内容是否出现？
对比V1/V2模型 → 是最新更新引起的回退？
AUC下降>0.03且连续2周 → 触发重训练
auc = roc_auc_score(y_true_redteam, y_scores) 		行业对标：成熟内容安全模型AUC通常>0.90；Google SafeSearch论文有参考；具体值为推断★
Review queue utilization
P3
（人审产能利用率）		 70-90% 健康区间
日波动<±10%		 >90% 或 <65% >95% 持续>2天
<50%（严重浪费）		 >95%：拆解原因
 日送审量↑ → 阈值被调低？内容激增？
 审核员人数↓ → 离职/请假？
 同时看IAA → 质量是否已下降？
<65%：
 阈值太高 → 看漏审率是否上升
 DAU下降 → 内容总量减少
utilization = daily_queue / (n_reviewers × cases_per_day) 		70-90%为运营管理通用最佳实践（客服中心/内容审核均适用）；Meta公开其审核团队规模约15,000人
Hash match latency P99
P0 法律
（CSAM检测延迟）		 P50: <50ms
P95: <200ms
P99: <500ms
（内容发布前完成）		 P99 >1秒 P99 >5秒
任何内容未检测直接发布		 P99突增：拆解流水线各阶段
 hash计算慢 → CPU/GPU负载？
 数据库查询慢 → 索引问题？DB连接池耗尽？
 网络延迟 → 跨区域请求？
P99过高的危险：内容在检测完成前已发布
→ 补救：发布后检测到再删除（but用户已看到）
目标：检测在内容首次曝光前完成
latency = hash_match_complete_time - upload_start_time 		PhotoDNA官方文档：毫秒级延迟；<100ms为行业实践目标★；内容上传到发布通常有数秒窗口
